# Design Reference Brief

## Purpose

Use this file to turn inspiration into design decisions.

References are not decoration. They help the team choose interaction patterns, hierarchy, density, tone, and anti-patterns before implementation.

### Adjacent Products

Adjacent products solve a similar interaction problem, even if the domain is different.

Examples:

- A finance app can inspire trust and dense data display.
- A calendar app can inspire scheduling and revision flows.
- A note app can inspire low-friction capture.
- A coaching app can inspire tone and progress continuity.
